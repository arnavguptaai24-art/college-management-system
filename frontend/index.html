Institutional Context

The College Management System is designed specifically for the Department of Machine Learning at BMS College of Engineering, Bangalore. The system represents a digital transition from legacy file systems to an interactive RESTful schema. It offers secure credentials and automated metrics mapping.

Application Tech Stack

  • Backend System: Node.js & Express.js server providing RESTful endpoints.
  • Storage Engine: SQLite persistent database accessed via better-sqlite3 for synchronous operations.
  • Security Layer: JWT-based authorization header policies combined with bcrypt hashing.
  • Frontend Interface: SPA compiled with native ES6 classes, CSS variables, and dynamic CSS layouts.

Object-Oriented Programming (OOP)

Consistent with the original system design, the front-end features a structure mirroring Java classes. We employ ES6 classes to enforce logical encapsulation, modular inheritance, and type structures:

1. Encapsulation

Data models for Person, Student, and Faculty encapsulate attributes and formatting logic (like salary and marks rendering) into cohesive code units.

2. Inheritance

The base class Person defines common fields (id, name, created_at) which are inherited by Student and Faculty, avoiding code repetition.

3. Polymorphism / Extensibility

Objects are loaded through API responses and dynamically instantiated. They provide polymorphic helper methods that adapt output structures based on the target class type.
